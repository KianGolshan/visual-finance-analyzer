2. System Architecture

Frontend — Next.js 14 App Router (TypeScript)

src/app/page.tsx — Landing page: hero, annotation semantics chips, DropZone upload, redirect on success
src/app/analyze/[docId]/page.tsx — Main workspace: canvas + toolbar + analyze button + results panel + follow-up chat + PDF page navigator
AnnotationCanvas.tsx — Fabric.js v7 canvas (SSR-safe dynamic import); 5 drawing tools; forwardRef handle exposes getImageDataUrl, getAnnotations, undo, redo, clear
AnnotationToolbar.tsx — Tool selector (5 tools), color picker (4 colors), undo/redo, clear, annotation count badge
AnalysisPanel.tsx — Structured results: insight cards (per annotation), confidence badges, flagged risks list, clickable follow-up question chips, export/copy buttons
DropZone.tsx — Drag-and-drop or click upload; client-side validation before POST; error display
PDF rendering — pdfjs-dist loaded dynamically in useEffect; worker at /pdf.worker.min.mjs (copied from node_modules at npm dev/build); 1.5× scale viewport for high resolution

Backend — Next.js API Routes (Serverless)

POST /api/upload — Validates file, saves to ./tmp/uploads/{uuid}/original.{ext}, stores DocumentMeta in server-side Map, returns docId
GET /api/upload?docId= — Returns DocumentMeta for a given docId (used by analyze page on mount)
GET /api/file/[docId] — Reads file from disk, returns with correct Content-Type (serves PDF/image to frontend)
POST /api/analyze — Orchestrates Claude call; handles both initial analysis and follow-up chat (via conversationHistory flag)
services/claude.ts — analyzeDocument() + analyzeWithHistory(); constructs Anthropic messages array with image block + text block; parses JSON from response (handles markdown code block wrapping)
services/promptBuilder.ts — buildSystemPrompt() once; buildUserPrompt() per-request with compact annotation descriptions (ID, type, color, semantic meaning, bounding box)
services/imageProcessor.ts — extractBase64, extractMediaType, extractAnnotationsFromFabric, resizeImageForClaude (max 1568px)
Storage — Local filesystem in dev; server-side Map for metadata. Swap writeFile → S3 putObject and Map → Redis for production.

4. Visual Prompt Engineering — Annotation Color Semantics

Red — Focus

"Focus specifically on this value or metric"

Best for: circling a specific number, EPS figure, revenue total

Yellow — Extract

"Extract and summarize this section or table"

Best for: bounding box around a full table row, income statement section

Blue — Trend

"Analyze the trend or pattern being pointed to"

Best for: arrows along chart trend lines, period-to-period comparisons

Green — Question

"User-defined context — treat label as a question or instruction"

Best for: text labels like "YoY growth rate?" or "Why did this drop?"
How it works: The system prompt defines this color → semantic mapping once. Each annotation's color, type, bounding box coordinates, and optional text label are serialized into a compact one-line description and appended to the user message alongside the composite image. Claude uses spatial context (pixel coordinates) + color semantics to produce per-annotation insights.

5. Claude API Integration — Prompt Construction & Token Flow

System Prompt (sent once) — Defines role ("financial document analyst"), color → semantic mapping, JSON schema instruction. ~200 tokens. Never repeated in follow-ups.
↓
buildSystemPrompt() in promptBuilder.ts
User Message — Image Block — Canvas.toDataURL() → base64 PNG → resized to max 1568px wide → sent as image block with media_type: image/png
↓
resizeImageForClaude() in imageProcessor.ts — keeps images under 5MB
User Message — Text Block — "Analyze the annotated financial document. Page 3. Canvas: 900×650px." + compact annotation lines: ID:a1b2 Type:circle Color:red Meaning:"Focus specifically…" BBox:[left:120,top:200,w:80,h:45]
↓
buildUserPrompt(annotationMetadata) in promptBuilder.ts — no JSON whitespace, abbreviations for keys
Claude Response (JSON) — { annotations_processed, insights[ { annotation_id, annotation_type, region_description, extracted_value_or_trend, financial_insight, confidence } ], overall_summary, flagged_risks[], follow_up_questions[] }
↓
parseClaudeResponse() handles both raw JSON and markdown-wrapped ```json blocks
Follow-up Turns — Conversation history appended; annotated image included in every user turn for persistent visual context; system prompt not resent.
Token budget: Initial analysis max_tokens: 2048 (full InsightResponse JSON) · Follow-up max_tokens: 1024 (conversational) · Model: claude-sonnet-4-20250514 · Image resized to ≤1568px wide before encoding
